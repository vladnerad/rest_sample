Описание API
Взаимодействие с API происходит по URL: localhost:8080/api/users/
1. Получить список всех пользователей (без ролей):
GET localhost:8080/api/users/all
2. Получить пользователя и его роли:
GET localhost:8080/api/users/{login}
3. Удалить пользователя:
DELETE localhost:8080/api/users/{login}
4. Добавить пользователя (если не указана ни одна роль - будет применена ROLE_USER по умолчанию):
POST localhost:8080/api/users/
5. Редактировать пользователя (требуется прислать новую сущность в теле запроса,
рекомендуется для изменения login/password/name):
PUT localhost:8080/api/users/{login}
6. Редактировать список ролей пользователя (в теле запроса должен быть массив ролей):
POST localhost:8080/api/users/{login}

По умолчанию доступно 4 роли (у каждого пользователя может быть несколько ролей):
    ROLE_ADMIN      - администратор
    ROLE_USER       - пользователь
    ROLE_OPERATOR   - оператор
    ROLE_ANALYTIC   - аналитик

Помимо стардатных HTTP кодов состояния в API используется следующий тип ответа:
{
    "success": true
}

или

{
    "success": false,
    "errors": [
        "Invalid password"
    ]
}

, где поле "success" указывает на успех/неудачу выполнения операции,
в случае неудаче в ответе будет еще и массив ошибок "errors"

Пример тела PUT запроса для добавления пользователя:
{
    "login": "ivan",
    "pass": "ivaN1",
    "name": "vanya",
    "roles": ["ROLE_ADMIN", "ROLE_OPERATOR"]
}

Пример тела POST запроса для изменения ролей пользоваетля:
{
    "roles": ["ROLE_ADMIN", "ROLE_OPERATOR"]
}

Пароль пользователя должен содержать букву в заглавном регистре и цифру.
Поля пользователя login, pass, name обязательно должны быть заполнены.
